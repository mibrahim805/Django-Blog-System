BlogSystem2 - Beginner Friendly File Explanation
================================================

Is file ka maqsad yeh hai ke project ki har important file ka role simple language mein samjhaya jaye.
Agar aap bilkul beginner hain, to is document ko top to bottom parhain.


0) High-Level Samajh (Project ka overall flow)
----------------------------------------------
1. User browser se request bhejta hai (jaise /post_list/).
2. URL config decide karti hai kis view ko call karna hai.
3. View database se data leta hai (models ke through).
4. View template ko data ke sath render karti hai.
5. Template HTML bana kar user ko show karti hai.
6. Like/comment par Notification model me record banta hai.
7. WebSocket se live notification bhi push hoti hai.



1) Root Files
-------------

- db.sqlite3
  - Kyun bani: SQLite database file, jahan actual data save hota hai.
  - Iska kaam: Users, posts, comments, likes, notifications sab yahin store hote hain.
  - Beginner note: Is file ko manually edit mat karein.

- manage.py
  - Kyun bani: Django project ka command entry point.
  - Iska kaam: `runserver`, `migrate`, `test`, `createsuperuser` jese commands run karna.


2) Project Config Folder - BlogSystem2/
----------------------------------------

- BlogSystem2/__init__.py
  - Kyun bani: Python package marker.
  - Iska kaam: Folder ko package banata hai. Code nahi hai, bilkul normal.

- BlogSystem2/settings.py
  - Kyun bani: Pooray project ki central settings.
  - Iska kaam:
    - INSTALLED_APPS me apps register hoti hain (`blog`, `channels`, `daphne`, etc.)
    - DATABASES define hota hai.
    - TEMPLATES path set hota hai.
    - ASGI application set hai (WebSocket support ke liye).
    - CHANNEL_LAYERS configured hai (abhi InMemory).
    - Context processor add hai: `blog.context_processors.notifications_badge`
      taake unread notification count globally templates ko mil sake.
  - Beginner note: Agar project behavior global level par change karna ho to yahin hota hai.

- BlogSystem2/urls.py
  - Kyun bani: Project-level URL router.
  - Iska kaam:
    - `/admin/` -> Django admin
    - `/accounts/` -> login/logout urls
    - Baqi sab -> `blog.urls`
  - Beginner note: Yahan app URLs include ki jati hain, business logic nahi.

- BlogSystem2/asgi.py
  - Kyun bani: ASGI setup (HTTP + WebSocket dono handle karne ke liye).
  - Iska kaam:
    - HTTP requests ko Django ASGI app ko bhejna.
    - WebSocket requests ko `blog.routing.websocket_urlpatterns` par bhejna.
    - `AuthMiddlewareStack` se websocket me logged-in user milta hai.

- BlogSystem2/wsgi.py
  - Kyun bani: Traditional WSGI deployment entry point.
  - Iska kaam: Production servers (WSGI based) ke liye app object provide karna.
  - Beginner note: WebSocket ke liye asgi.py important hota hai.

- BlogSystem2/routing.py
  - Kyun bani: Optional project-level websocket routes ke liye jagah.
  - Current state: Empty file.
  - Beginner note: Is project me websocket routing app-level (`blog/routing.py`) me ho rahi hai.


3) App Folder - blog/
----------------------

- blog/__init__.py
  - Kyun bani: Python package marker.
  - Iska kaam: folder ko importable package banata hai.

- blog/apps.py
  - Kyun bani: Django app config.
  - Iska kaam: App ka naam aur default pk field define karta hai.

- blog/models.py
  - Kyun bani: Database tables ka structure define karne ke liye.
  - Iska kaam:
    - `Post`: title/content/author/timestamps/is_published
    - `Comment`: post + user + content
    - `Like`: user + post (ek user ek post ko 1 hi dafa like kar sakta hai)
    - `Notification`: kis user ko notification gayi, kis post ki, kis sender ne bheji
  - Important line concept:
    - `unique_together = ('user', 'post')` -> duplicate like prevent karta hai.
    - `related_name='comments'` -> post.comments se comments access hote hain.

- blog/forms.py
  - Kyun bani: HTML forms ko Django model/forms se bind karne ke liye.
  - Iska kaam:
    - `RegistrationForm` -> user signup
    - `PostCreateForm` -> title/content + `is_published` checkbox
    - `CommentForm` -> comment content

- blog/views.py
  - Kyun bani: Request handle karna, business logic chalana, response dena.
  - Iska kaam:
    - `UserRegisterView` -> register
    - `PostListView` -> published posts list + liked_post_ids context
    - `PostCreateView` -> post save + publish hone par sab users ko notification
    - `CommentCreateView` -> comment save + post author ko notification
    - `LikeView` -> like/unlike toggle + author ko notification
    - `notifications_list` -> notifications page show + unread ko read mark
    - `notifications_unread_count` -> JSON me unread count
    - `create_and_push_notification(...)` -> reusable helper:
      DB me notification create karta hai aur websocket par push karta hai.

- blog/urls.py
  - Kyun bani: App-level routes define karne ke liye.
  - Iska kaam:
    - register, create_post, post_list
    - comment create endpoint
    - like toggle endpoint
    - notifications page
    - unread count API

- blog/consumers.py
  - Kyun bani: WebSocket message handling ke liye.
  - Iska kaam:
    - `NotificationConsumer` user ko uski private group `user_<id>` me add karta hai.
    - View se jab `group_send` hota hai to consumer browser ko message bhejta hai.
  - Beginner note: Ye realtime feature ka core hai.

- blog/routing.py
  - Kyun bani: WebSocket URL patterns define karne ke liye.
  - Iska kaam: `/ws/notifications/` ko `NotificationConsumer` se bind karta hai.

- blog/context_processors.py
  - Kyun bani: Har template ko common data dene ke liye.
  - Iska kaam:
    - Agar user authenticated hai to unread notifications count nikalta hai.
    - `unread_notifications_count` ko global template context me bhejta hai.

- blog/admin.py
  - Kyun bani: Django admin panel custom display ke liye.
  - Iska kaam:
    - `Post` admin me register hai.
    - list page me `title` aur `content` dikhata hai.

- blog/tests.py
  - Kyun bani: Feature checks automated karne ke liye.
  - Iska kaam (current tests):
    - like/unlike test
    - comment create test
    - like se notification create hoti hai test
    - unread count + mark-read flow test


4) Migrations - blog/migrations/
--------------------------------

- blog/migrations/__init__.py
  - Kyun bani: migrations package marker.

- blog/migrations/0001_initial.py
  - Kyun bani: first migration file (initial tables banane ke liye).
  - Iska kaam: Post, Notification, Comment, Like tables create karta hai.
  - Beginner note: Is file ko normally manual edit nahi karte; `makemigrations` se banti hai.


5) Templates Folder
-------------------

- templates/base.html
  - Kyun bani: common layout (master template).
  - Iska kaam:
    - Navbar include karta hai.
    - `{% block content %}` area deta hai child templates ke liye.
    - JS ke through websocket connect karta hai.
    - live toast show karta hai.
    - unread count sync endpoint hit karta hai.
    - socket close hone par reconnect try karta hai.

- templates/includes/navbar.html
  - Kyun bani: reusable navbar partial.
  - Iska kaam:
    - Home link
    - Notifications bell + unread count (`unread_notifications_count`)

- templates/post/post_list.html
  - Kyun bani: posts listing screen.
  - Iska kaam:
    - all published posts show
    - Like/Unlike button
    - Add Comment link
    - likes/comments counts
    - recent comments preview

- templates/post/post_create.html
  - Kyun bani: post create form screen.
  - Iska kaam:
    - form render karta hai (`title`, `content`, `is_published`)
    - submit par post create hota hai.

- templates/comments/comment_create.html
  - Kyun bani: comment create screen.
  - Iska kaam: selected post ke liye comment save karwana.

- templates/notifications/list.html
  - Kyun bani: notifications history page.
  - Iska kaam: user ki notifications list show karta hai.

- templates/registration/login.html
  - Kyun bani: login form page.
  - Iska kaam: Django auth login form render.

- templates/registration/register.html
  - Kyun bani: custom signup page.
  - Iska kaam: user registration form render.


6) __pycache__ Folders
----------------------
- Kyun bante hain: Python runtime compiled `.pyc` cache files generate karta hai.
- Iska kaam: code execution ko thoda fast banana.
- Beginner note: In files ko edit/delete karne ki normal zarurat nahi hoti.


7) Notification Feature ka Simple End-to-End Flow
--------------------------------------------------
1. User B likes/comments on User A post.
2. `LikeView` / `CommentCreateView` me `create_and_push_notification` call hota hai.
3. Notification database me save hoti hai.
4. Same waqt websocket group `user_A_id` par message push hota hai.
5. User A ke browser ka `base.html` JS message receive karta hai.
6. Bell count increment hota hai + toast popup show hota hai.
7. User A `/notifications/` page open karta hai to unread read mark ho jati hain.


8) Beginner Ke Liye Practical Tips
----------------------------------
- Naya model add karo -> `models.py` -> `makemigrations` -> `migrate`
- Naya page add karo -> `views.py` + `urls.py` + template file
- Form field add karo -> `forms.py` and template me `{{ form.as_p }}` hone par auto render
- Realtime issue aaye to check list:
  1) websocket URL
  2) ASGI setup
  3) consumer group name
  4) browser console errors
  5) notification DB row create ho rahi hai ya nahi


9) Short Conclusion
-------------------
Yeh project Django + Channels based blog system hai jisme posts, comments, likes aur realtime notifications hain.
Agar aapko code samajhna start karna ho to best order yeh hai:
`urls.py -> views.py -> models.py -> templates -> consumers.py`.
